bettercap嗅探与arp污染

发表于 2022-05-17 更新于 2022-07-16

基本用法

help 查看当前开启的模块

get * 查看当前所有配置

get net.sniff.* 查看net.sniff模块的所有配置

set net.sniff.output [属性值] 设置模块的属性

help net.sniff 查看模块的帮助文档

net.sniff on 开启某模块

net.sniff off 关闭某模块

net.recon模块：用于发现网内主机，基本信息有IP地址、MAC地址等

​ net.show：net.recon模块的一个功能，列出当前发现的所有主机信息

网络嗅探

set net.sniff.output ceshi.pcap 设置导出包

set net.sniff.regexp baidu.com 设置关键字

net.sniff on 开启网络嗅探，此时用户访问百度我们就可以收到http包

net.sniff off 关闭网络嗅探

arp污染

arp.spoof on 开启arp污染

arp.ban 劫持arp流量但不放包，使全局断网

set arp.spoof.targets 192.168.1.1 只针对特定IP进行arp污染

DNS欺骗

set dns.spoof.domains www.sina.com 设置要欺骗的域名

set dns.spoof.address 127.0.0.1 设置将欺骗域名解析到我们自己的恶意IP

dns.spoof on 开启dns欺骗

信息收集思路

发表于 2022-04-13 更新于 2022-07-16 分类于 1.信息收集

【组织架构】

【主域名】

–备案反查顶级域名（http://icp.chinaz.com）

–ip反查域名

–组织架构，子公司信息收集

【子域名】

–子域名爆破（非泛解析域名；收集subfinder、Sbulist3r、OneForAll、子域名挖掘机、ksubdomain的字典）

–子域名查询（theHarvester、virustotal、ce.baidu.com、url.fht.im……水泽……，集成水泽的所有功能，并写一个接口测试模块，信息收集开始前，显示有效接口列表；信息收集结束后，如果有未正常结束的，产生告警。并对所有接口模块化编程，使用户在知道哪个接口异常后，可以轻松找到对应接口的脚本文件，纠错，并使用该单一脚本文件补全结果。）

–IP反查域名

–友链爬虫

‘’‘算法：后爬到的域名越靠前，证明它越难被收集到，越有可能测到漏洞’‘’

‘’‘算法：多列表切割结果，实时存放结果，如一个列表存放10000个子域名，10000满后就将列表中的值全部保存到文件1中，依次类推，直到结束后，从编号最大的一个文件开始倒序读取子域名，存入同一文件当中，这样最后收集到的子域名就变成了最终子域名列表的第一个’‘’

【IP:端口】

–子域名解析

–nmap

  • 收集IP列表及其端口开放情况，批量对某些可以公开登录的服务进行弱密码爆破

【github信息收集】

–水泽

  • zeebe密钥泄露
  • 各类服务、主机用户名密码泄露
  • 源代码泄露
  • Sql代码泄露

【存活URL验证】

–http/https双验证

–vpn二次验证（双验证不存活，则挂三个不同vpn二次验证，还不存活则认定不存活）

【存活URL版本特征检索】

–目录爬虫（根据某些cms的特殊目录名，如swagger-ui、zentao、seeyon等来确定）

–响应头检索（根据Server、x-forwared-for等响应头泄露的信息来判断）

–报错信息判断（根据nginx、apache、thinkphp等的特征报错信息来判断；可以使用奇怪文件名、目录回溯、错误参数等方法来引发报错）

sqlmap使用总结

发表于 2022-02-18 更新于 2022-07-16 分类于 1.工具总结 ， 1.web安全工具

基础sqlmap指令：

常用参数：

–timeout= 设置http在几秒后超时

–ignore-timeouts 忽略http超时

–output-dir=目录 设置sqlmap结果的输出路径

c:\users\用户名\AppData\Local\sqlmap\output\ 删除sqlite即清除缓存重新扫描

–test-filter=mypayload1 指定使用某payload

–test-skip=mypayload1 跳过某payload

–force-ssl 告诉sqlmap测试的链接是https协议

–technique=TECH SQL注入技术测试（默认BEUST）

–start：指定开始的行（跑指定行数据）

–stop：指定结束的行
